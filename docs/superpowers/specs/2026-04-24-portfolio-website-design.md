## 一、專案目標

建立一個個人作品集靜態網站，用於求職與接案兩種場合：

- **求職**：面向技術背景受眾（工程師、技術主管），展示技術深度與問題解決能力
- **接案**：面向非技術背景受眾（中小企業、個人客戶），展示成效與流程自動化價值
- **作品集本身**即為一個展示工程能力的作品

---

## 二、品牌定位

**核心定位：** AI × 流程自動化

**設計語言：** 精準、有效率、有品味，不炫技

**視覺風格：**
- 深色底（深色背景比白底對工程師更具信任感，且更凸顯截圖與數字）
- 單一強調色，選擇工業感色調（推薦：電藍 `#3B82F6` 或深琥珀 `#D97706`）
- 避免常見的紫色漸層，目標是有記憶點

**Hero 定位句（推薦版本）：**
> 「我幫你把每天重複做的事，變成自動跑的系統。」
> Java 後端 × Python × AI 整合 × 流程自動化

**備用版本：**
> 「你的時間值得更好的用法。」
> 我用 AI 和自動化，幫你把流程從手動變自動。

---

## 三、技術架構

### 框架
**Astro**（靜態網站生成器）

選擇理由：
- 預設零 JavaScript，頁面載入極快，SEO 友善
- 支援 React / Vue 元件混用
- 內建 Content Collections，作品以 Markdown 管理，新增作品只需新增一個 `.md` 檔
- View Transitions API 內建支援，頁面切換質感提升

### 部署
- **平台：** Vercel（免費方案）
- **Domain：** 建議綁定自訂 domain（如 `marschen.dev`）

### Astro 特性清單
| 功能 | 用途 |
|------|------|
| Content Collections | 管理 `/projects` 作品資料 |
| View Transitions API | 頁面切換動畫，提升整體質感 |
| OG Image 自動生成 | 每個作品頁自動產生 Open Graph 圖片，分享到 LinkedIn 或傳給客戶時連結預覽即顯示標題與成效數字 |
| 圖片最佳化 | 截圖自動壓縮、轉 WebP、lazy load |

## 四、頁面架構

### `/ 首頁`
```
├─ Hero 區塊
│   ├─ 定位句（「我幫你把每天重複做的事，變成自動跑的系統。」）
│   ├─ 副標（技術能力標籤：Java 後端 × Python × AI 整合 × 流程自動化）
│   └─ 兩個 CTA 按鈕：「看作品」/ 「聯絡我」
├─ 一句話價值說明（「我解決什麼問題」，不是職稱）
├─ 技術標籤概覽（Python / Java / Google Apps Script / VBA / AI 整合）
├─ 作品卡片列表
│   ├─ 成效數字（視覺優先，最大）
│   ├─ 一句話摘要
│   ├─ 產業標籤（餐飲業 / 醫療 / 個人財務⋯）
│   └─ 技術標籤（次要位置，較小）
└─ 底部 CTA：「有類似需求？30 分鐘免費聊」
```

### `/projects/[slug] 個別作品頁`
```
├─ 1. 標題 + 一句話成效（結論優先，BLUF 原則）
├─ 2. 問題背景（故事化描述，非技術受眾也看得懂）
├─ 3. 解決方案（說明做了什麼，附截圖或 GIF）
├─ 4. 技術決策（為什麼選這個技術，不選那個）
├─ 5. 挑戰與踩坑（遇到什麼問題、怎麼解決）
├─ 6. 成效數字區塊（進度條 + 放大數字，視覺衝擊）
├─ 7. GitHub 連結（選填）
└─ 8. 底部 CTA：「有類似需求？找我談談」
```

### `/about 關於我`
```
├─ 自我介紹（第一段說「我幫誰解決什麼問題」）
├─ 轉職故事（敘事型，不是條列型）
│   重點：從 PM 看著工程師解決問題 → 自己成為能解決問題的人
│   核心句：「找出問題、優化流程、提高效率——這件事在我當 PM 時就讓我有成就感，
│           成為工程師之後，我終於能親手把解法做出來。」
├─ 工作經歷時間軸（每段加「關鍵學到什麼」註記）
├─ 技術能力標籤
└─ 底部 CTA：「想找一個懂業務也懂技術的人？」
```

### `/contact 聯絡頁`
```
├─ 一句話引導（「告訴我你的問題，我來評估能不能幫你」）
└─ 極簡表單：姓名 + Email + 需求描述（三欄）
```

## 五、資料結構

### Content Collections 路徑
`src/content/projects/[slug].md`

### Frontmatter 欄位定義

```yaml
---
title: "工具名稱"
summary: "一句話成效描述（作品卡片顯示）"
industry: ["產業標籤1", "產業標籤2"]   # 產業分類
tech: ["Python", "CLI"]               # 技術標籤
metric: "節省 55 分鐘／週"             # 成效數字，視覺化大字顯示
github: "https://github.com/..."      # 選填
demo: ""                              # 選填，Demo 影片或連結
cover: "./images/[slug]-cover.png"    # 封面截圖
order: 1                              # 首頁排序
---
```

### Markdown 正文結構（固定四段）

```markdown
## 問題背景
（故事化描述，非技術受眾也看得懂）

## 解決方案
（說明做了什麼，附截圖或 GIF）

## 技術決策
（為什麼選這個技術，不選那個）

## 挑戰與踩坑
（遇到什麼問題、怎麼解決）
```

| `video-notes` | 影片轉錄文字工具 | 醫美教育、內容創作 | Python、AI、Whisper |

## 六、成效視覺化設計

在 `/projects/[slug]` 的成效區塊，採用「進度條 + 放大數字」格式：

```
節省作業時間
████████░░  70%
```

- 不使用表格
- 數字放大到標題級別
- 目標：掃過去就記住，不需要閱讀
