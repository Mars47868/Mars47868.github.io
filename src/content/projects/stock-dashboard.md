## 問題背景

透過多家券商持有美股，每次想了解整體損益，流程都一樣麻煩：逐一登入各帳戶記下持倉，最後手動加總、換算台幣再比較。

這個過程穩定耗費半小時，而且算完仍然看不出配置是否合理——有些標的長期抱著、漲幅很小，資金就靜靜躺著，沒有一個地方可以快速看出哪支股票的錢在有效工作。

## 解決方案

分兩層架構：**GAS 後端**負責從 Google Sheets 讀取交易記錄、計算損益、抓取股價；**靜態前端**部署到 Cloudflare Pages，透過 API 呼叫後端取得資料。

前端為 4 個 tab 的儀表板：

- **總覽**：今日損益橫幅（正綠負紅）、8 張 KPI 卡片（現金淨投入、持倉成本、市值、已/未實現損益、MTD、YTD、活躍持倉數）、成員摘要列
- **持倉**：依券商分頁、持倉明細表（含 CAGR、近 30/90 天動能%、狀態標籤）、已實現損益表
- **績效**：淨值走勢折線圖
- **觀察清單**：追蹤中標的，首次點入才 lazy load

持倉熱度排行依近 30 天漲幅把持倉分成四個等級（強勢、上漲、盤整、下跌），一眼看出哪些標的在積極貢獻報酬、哪些資金長期沒有動靜。

持有人篩選器讓多人共用帳戶的情境下，能快速切換查看特定成員的指標，篩選直接聚合記憶體中的既有資料，不需重打 API。

## 技術決策

**前後端分離**：GAS Web App 作為後端 API，前端靜態 HTML/JS 部署到 Cloudflare Pages。主要好處是前端有固定的 HTTPS URL、載入速度不受 GAS 執行時間影響，且前端可以單獨迭代。

**股價改用 Yahoo Finance API**：Google Sheets 內建的 `GOOGLEFINANCE()` 只能在試算表公式中使用，無法在 GAS 程式邏輯中呼叫，改用 Yahoo Finance 的非官方 API 抓取，配合多層快取策略（現價 15 分鐘、昨收價與歷史資料 24 小時）減少 API 呼叫次數。

**MTD/YTD 公式選擇**：採用 `ΔPortfolioValue − 淨現金流入` 而非 `當前股數 × 期間價差`。月中若有加減倉，後者會算出錯誤數字；前者在任何時間點加減倉後都能精準反映該期間的真實績效。

**存取控制在後端過濾**：儀表板部署在公開的 Cloudflare Pages，不同成員需要只看自己的持倉。前端過濾是假安全——資料已整包傳到瀏覽器，DevTools 可見。設計成後端在 `getTransactionData()` 取得全部交易後立即按身分過濾，後續所有計算（FIFO、KPI、MTD/YTD）皆基於過濾後資料，後端自然只回傳有權限的部分。憑證採無狀態 HMAC 簽章（30 天效期），儲存於 Script Properties，不進程式碼或 git。

## 挑戰與踩坑

**持有人篩選器的效能設計**：篩選總覽、持倉兩個 tab 時，前端已握有完整的逐人資料，直接聚合即可，不需重打 API，切換是瞬間的。但績效圖的歷史每日市值需後端按持有人重算，若每次切人都重抓歷史 K 線會很慢。解法是把後端拆成兩層：K 線層的快取 key 不含持有人（切人共用快取不重抓），聚合層才吃持有人參數，快取 key 加上持有人簽章。

**批次股價抓取的速率限制**：Yahoo Finance API 非官方，批次抓取多支股票時若呼叫過於頻繁會被封鎖。透過 CacheService 分層快取大幅減少實際 API 呼叫次數，並在 `clearStockPriceCache()` 提供完整清除機制，讓使用者在需要時強制刷新。
