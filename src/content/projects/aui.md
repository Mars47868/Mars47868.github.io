## 問題背景

擔任 PM 期間，每週固定要整理一份庫存彙整表在部門週會上使用。流程是這樣的：先從 ERP 匯出三份報表，每份格式都不同；再逐欄手動比對多個倉別的數字——存貨彙總、銷售訂單、出貨退貨，全部得手動對齊、相互驗證。

整個流程穩定耗費一小時。最折磨的不是時間，而是每次做完都提心吊膽——怕哪個數字算錯，怕哪個倉別搞混。這不是一次性的任務，是每週都要重複的負擔。

## 解決方案

把三份 ERP 報表放進指定資料夾，執行一行指令：

```shell
aui.exe 20991230
```

五分鐘後拿到格式正確、欄位齊全的彙整表，可以直接帶進會議室。程式打包成 `.exe`，連 Python 都不用安裝，同事也能直接執行。

## 技術決策

問題的本質是「格式轉換加欄位映射」——重複且有規則，可以被程式化。

選 Python 而非 Excel VBA，主要有三個原因：一、可以放進 Git 做版本控制，欄位規則怎麼改都有記錄；二、打包成 exe 讓非工程師同事不需要任何開發環境就能執行；三、日後若需要排進 CI 自動跑，Python 比 VBA 更容易整合。

倉別的欄位映射規則統一放在外部設定檔維護，不寫死在程式碼中。以後倉別增減，只需改設定檔，不需要動程式邏輯。

## 挑戰與踩坑

最大的挑戰是不同 ERP 報表的欄位命名和日期格式不統一。同樣是「出貨日期」，不同報表可能叫不同名字，日期格式也可能是三種不同寫法。如果程式直接信任輸入，遇到格式變動就會靜默算錯，比手動整理更危險。

解法是定義欄位映射規則加格式驗證：程式在讀取每份報表時，先比對預期欄位清單，若發現不符合就立刻報錯並指出哪份來源，強迫使用者確認，而不是讓錯誤悄悄流入輸出。
